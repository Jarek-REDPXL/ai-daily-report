- **Reactive CSS via live custom properties (JS-shim branch)** → Adam Argyle's 'Prop For That' (nerdy.dev, Jun 13 2026) exposes runtime browser data (input values, form validity, pointer/scroll, media progress, image colors) as live `--live-*` CSS custom properties through a single `data-props-for` attribute — moving slider-fill / validity-gating / pointer effects out of per-frame JS into the stylesheet. Newly practical because Style Queries reached all major browsers (Baseline); a JS shim drives it, so it's a dependency to audit, NOT a native primitive — distinct from the native modern-CSS-primitives thread. A few signals (network/battery/geo) are Chromium-only → guard with `var()` fallbacks. Card: card-webdesign-prop-for-that. - **Scroll-TRIGGERED animations (the IntersectionObserver-in-CSS branch)** → Chrome shipped scroll-triggered animations (Chrome 145/146, 2026): a normal timed `@keyframes` animation that plays ONCE when an element crosses a viewport threshold (vs scroll-DRIVEN, which scrubs to scroll position). Two new properties: `timeline-trigger: --t view() entry 100% exit 0%` (when) + `animation-trigger: --t play-forwards` (bind); actions `play-forwards`/`play-backwards`/`play-once`/`pause`/`reset`/`replay`/`none`. Deletes reveal-on-scroll JS for most traffic. **Chromium-only** (Chrome's own demo ships an `IntersectionObserver` fallback; spec still settling) → progressive enhancement, NOT a dependency you delete for all users. Card: card-webdesign-scroll-triggered-animations (Chromium-only branch of thread-modern-css-primitives). Watch for 2nd engine → Baseline. (2026-06-21)

## Durable lessons (the compounding edge — evergreen takeaways we've taught)
<!-- promote a thread's conclusion here once it's a stable, reusable principle. -->
- New CSS visual primitives (corner-shape, scroll-driven animation, anchor positioning) that **degrade gracefully** can ship the day they hit one engine — pure upside, no `@supports` gymnastics, because unsupported browsers fall back to the old rendering. Lead with these.
- **Baseline vs Chromium-only is the deploy decision, not the existence of the feature.** A *Chromium-only* primitive (carousel, sibling-index) ships today only as decorative progressive enhancement. A *Baseline* one (popover + invoker commands + anchor positioning) can **replace a JS dependency outright** for all users — flag which branch a feature is on before recommending "ship it."
