# The AI Edge — Predictions Ledger

> **Accountability is what makes the briefing trustworthy and self-improving.**
> When a report makes a forward-looking call, log it here. Each run checks whether
> any open prediction resolved and records the outcome. Being publicly right (or
> honestly wrong) is the edge a generic news feed can't fake.

## How to use this file (instructions for each run)
1. **Before writing:** scan "Open predictions" — did anything resolve in the last
   ~24h? If so, move it to "Resolved" with the outcome (✅ held / ❌ missed /
   ⚖️ partial) and a one-line note, and mention the resolution in today's report.
2. **After writing:** if today's report makes a concrete, checkable forward call,
   add it to "Open predictions" with a date and (if possible) a rough due window.
3. Keep predictions **falsifiable** (a date, a number, a shipped/not-shipped event).
   Vague vibes don't belong here.
4. **Tag each prediction with its domain** using a `[domain]` marker (valid slugs
   in [`../DOMAINS.md`](../DOMAINS.md)) right after the date, so the brain can
   filter the ledger per discipline going forward.
