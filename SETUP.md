# Setup — get The AI Edge live + running on autopilot

Three short stages. Stage 1 gets it on GitHub (private). Stage 2 turns on the
automatic daily + weekly reports. Stage 3 (optional) puts the site online.

---

## Stage 1 — Push to your private GitHub repo  (~2 min)

From inside the unpacked `ai-daily-report` folder:

```bash
./PUSH_TO_GITHUB.sh
```

- If you have the **GitHub CLI** (`gh`), it creates a **private** repo
  `redpxl/ai-daily-report` and pushes automatically.
- If not, it prints a 3-line manual fallback: create an empty private repo at
  https://github.com/new (name `ai-daily-report`, **Private**, no README), then
  add the remote and `git push -u origin main`.

> Already restored from the `.bundle`? Same thing — just run the script.

---

## Stage 2 — Turn on automatic daily + weekly reports  (~3 min)

The reports are generated by **GitHub Actions** — they run on GitHub's servers
on a schedule, so **nothing on your computer needs to be on**.

1. **Add your Anthropic API key as a secret.**
   - Get a key: https://console.anthropic.com → API Keys.
   - In the repo: **Settings → Secrets and variables → Actions → New repository
     secret**.
   - Name it exactly `ANTHROPIC_API_KEY`, paste the key, save.

2. **Allow Actions to write to the repo.**
   - **Settings → Actions → General → Workflow permissions** →
     select **“Read and write permissions”** → Save.

That's it. The two schedules are already defined:

| Workflow                | File                                   | Default schedule (UTC)        |
|-------------------------|----------------------------------------|-------------------------------|
| Daily AI Briefing       | `.github/workflows/daily-report.yml`   | `0 7 * * *`  (07:00 daily)    |
| Weekly AI Briefing      | `.github/workflows/weekly-report.yml`  | `0 17 * * 0` (Sun 17:00)      |

- **Test it now without waiting:** repo **Actions** tab → pick *Daily AI
  Briefing* → **Run workflow**. In a couple of minutes a new commit appears with
  today's report, and the site shows it.
- **Change the time / timezone:** edit the `cron:` line. Cron is in **UTC**.
  You're in Poland (CEST = UTC+2 in summer): `0 7 * * *` fires at **09:00**
  your time. For 08:00 local in summer use `0 6 * * *`.

> Cost: each run uses Claude API tokens billed to your Anthropic account. A daily
> report is small; monitor usage at console.anthropic.com.

---

## Stage 3 — (Optional) Put the site online

By default the site is **local**: run `./serve.sh` and open
`http://localhost:8099`.

To also host it live (private repos on GitHub Pages require a paid plan; public
repos are free):

1. **Settings → Pages → Source: “GitHub Actions”.**
2. The included `.github/workflows/deploy-pages.yml` publishes on every push to
   `main`. Your site lands at:
   `https://redpxl.github.io/ai-daily-report/`

---

## Daily routine (yours)

Nothing required. Open the site whenever you like:
```bash
cd ai-daily-report && ./serve.sh
```
New reports appear at the top automatically; older ones live in the tab bar.
Download any week's PDF with the button on its report.
